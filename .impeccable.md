## Design Context

### Users

Developers and technical users managing GitHub repository backups. They interact with Git Plus through a web dashboard to configure source connections, manage tokens, and control repository sync settings. They value efficiency, clarity, and tools that feel native to their workflow.

### Brand Personality

**Modern, Friendly, Polished** — approachable SaaS feel that's refined but not cold. The interface should feel like a well-crafted developer tool that respects the user's time and intelligence.

**Emotional goals**: Confidence (things are working), calm (nothing is overwhelming), competence (the tool is professional-grade).

### Aesthetic Direction

**Visual tone**: Clean, component-driven, modern defaults. Crisp layouts with purposeful spacing.

**References**:

- Linear / Raycast — tight spacing, keyboard-first UX, refined dark surfaces
- Tailwind UI / shadcn — component-driven aesthetic, utility-inspired, modern defaults

**Anti-references**: Cluttered enterprise dashboards, overly decorative designs, raw JSON/config editors.

**Theme**: Light mode is the primary experience. Dark mode should be supported but light mode gets polish priority.

**Color system**: Zinc-based neutral palette (current). Professional and restrained — accent colors used sparingly for status and actions.

### Design Principles

1. **Clarity over cleverness** — Every element should earn its place. Prefer explicit labels and visible hierarchy over hidden menus and clever layouts.
2. **Density with breathing room** — Show useful information without overwhelming. Use spacing deliberately — not too sparse (wastes screen), not too packed (cognitive overload).
3. **Component-native feel** — Lean into Mantine's design language. Consistency with the component library is more important than custom flourishes.
4. **Status at a glance** — Users should understand system state (sources configured, tokens valid, sync status) without clicking into detail views.
5. **Progressive disclosure** — Show the essential first, reveal complexity on demand. CRUD actions in context menus, advanced options in modals.

### Technical Design Tokens

- **Font**: Geist (sans) / Geist Mono (mono)
- **Primary color**: Zinc palette
- **Default radius**: sm (8px)
- **Spacing scale**: Mantine defaults (xs: 10px, sm: 12px, md: 16px, lg: 20px, xl: 24px)
- **Icon library**: Tabler Icons (14-20px)
- **UI library**: Mantine v9
